## Description

NestJS + TypeORM + PostgreSQL + Elasticsearch 기반 블로그 API 서버입니다.

- 표준 응답: `@3xhaust/nest-response`
- 포스트 조회/작성/수정
- 태그 목록/태그 필터
- 관련 포스트 추천
- 비로그인 댓글 작성(랜덤 닉네임)
- 관리자 전용 답글
- 하트(클라이언트당 1회)

## Environment

`.env.example`를 복사해서 `.env`를 생성하세요.

```bash
cp .env.example .env
```

관리자 로그인은 `.env`의 `ADMIN_NICKNAME`, `ADMIN_PASSWORD`를 사용합니다.

JWT + 쿠키 기반 세션 동작:

- `access_token`(단기) 만료 시 클라이언트가 `POST /auth/refresh` 호출
- `refresh_token`으로 access 재발급 후 기존 요청 자동 재시도
- 쿠키는 `httpOnly`로 저장

### Markdown 파일 가져오기

마크다운 파일(또는 폴더)을 블로그 `posts` 스키마로 변환해서 DB에 업서트할 수 있습니다.

```bash
# 기본: ../velog-series 폴더를 읽어 업서트
npm run import:markdown

# 특정 파일/폴더 지정
npm run import:markdown -- ../velog-series/00-start.md

# DB 반영 없이 변환만 (JSON 파일 출력)
npm run import:markdown -- ../velog-series --dry-run --out=./tmp/imported
```

지원 형식:

- Frontmatter: `title`, `slug`, `tags`, `thumbnail`, `readingTime`
- 본문 블록: heading(`#~###`), paragraph, quote(`>`), divider(`---`/`***`), image(`![alt](url)`), code fence
- interactive 코드 fence: ````interactive jsx editable=restricted lines=2,3````

## Project setup

```bash
$ npm install
```

## Compile and run the project

```bash
# development
$ npm run start

# watch mode
$ npm run start:dev

# production mode
$ npm run start:prod
```

서버 기본 주소는 `http://localhost:3000` 입니다.

## Run tests

```bash
# unit tests
$ npm run test

# e2e tests
$ npm run test:e2e

# test coverage
$ npm run test:cov
```
